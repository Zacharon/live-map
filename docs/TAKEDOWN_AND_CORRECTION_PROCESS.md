# Takedown and Correction Process

Live Map needs correction handling because public sources can be wrong, revised, disputed, or retracted.

Process:

1. Record the source, event ID, requester, reason, and affected URL.
2. Verify whether the upstream provider corrected or removed the item.
3. Mark affected events `corrected`, `disputed`, or `retracted` when appropriate.
4. Preserve an audit note without exposing private requester data.
5. Remove or reduce precision for sensitive personal, humanitarian, health, or security information when justified.

Corrections must not be hidden by later ingestion runs.
