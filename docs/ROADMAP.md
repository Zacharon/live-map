# Roadmap

## Phase 1

- Modular app foundation
- Five dashboard modes
- 45+ layer registry
- CII prototype
- 92-exchange finance registry
- Event-to-market correlation prototype
- Local alert-rule preview
- API scaffolding
- Security/privacy documentation

## Phase 2

- Master OSINT source registry and Source Explorer governance foundation
- Source access, licensing, quality, verification, cache, retention, and takedown policy docs
- Aviation and maritime provider integrations
- Cyber and infrastructure provider adapters
- Static infrastructure GeoJSON with attribution
- Scaffold source: `src/data/scaffolded-modules.js`

## Phase 3

- Provider-agnostic AI briefs with citations, prompt-injection controls, and local Ollama option

## Phase 4

- OSINT resource directory by country using the governed source registry rather than a copied third-party directory

## Phase 5

- Investigations workspace and network visualization

## Phase 6

- Compliance and case-management models with audit trails

## Deep/dark web and restricted-source boundary

Only lawful, licensed, customer-authorized, or public sources are in scope. The project must not implement unlawful access, credential theft, malware deployment, raw password display, restricted scraping, or authentication bypass.
