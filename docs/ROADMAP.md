# Roadmap

## Phase 1

- Modular app foundation
- Five dashboard modes
- 45+ layer registry
- CII prototype
- 92-exchange finance registry
- Event-to-market correlation prototype
- Local alert-rule preview
- API scaffolding
- Security/privacy documentation

## Phase 2

- Master OSINT source registry and Source Explorer governance foundation
- Source access, licensing, quality, verification, cache, retention, and takedown policy docs
- Completed foundation: USGS, NASA EONET, NOAA/NWS, GDACS, Source Explorer, provider health diagnostics, taxonomy, grouping, saved views, URL filters, incident clustering, synchronized map layers, and mobile-friendly feed controls.
- Phase 2A implemented: ReliefWeb humanitarian adapter foundation, CISA KEV non-geographic cyber events, NVD focused CVE enrichment, provider schedules, request budgets, and non-geographic event handling.
- Phase 2B implemented foundation: SEC EDGAR material filing classification, FRED macro observations, EIA energy observations, provider incremental state, and finance/commodity dashboard scaffolding.
- Phase 2C implemented: official status/RSS feeds, GDELT discovery leads, deeper provider state boundaries, and open news/infrastructure foundations.
- Phase 2D implemented foundation: country navigation, `/countries`, CII v2 distribution controls, diagnostics separation, provider capability registry, provider scaffold generator, and independently gated open RSS provider groups.
- Phase 3 implemented foundation: consumer navigation, Standard/Advanced modes, collapsible drawers, onboarding, presets, global search, moving-object API, OpenSky boundary, Global Fishing Watch boundary, AISHub boundary, and aviation/maritime reference registries.
- Strategic chokepoints v1 implemented: generalized registry, deterministic event correlation, explainable local condition assessment, map layer, detail pane, URL state, and cross-domain display fields. Future provider work remains separately governed by `docs/STRATEGIC_DATA_SOURCE_ROADMAP.md`.
- Later provider waves: ACLED, aircraft tracking, maritime tracking, and licensed finance/OSINT feeds after credentials, licensing, rate limits, and safety reviews are complete.
- Cyber and infrastructure provider adapters beyond CISA KEV/NVD remain planned until each source has a registry entry, budget, cache policy, and safety notes.
- Static infrastructure GeoJSON with attribution
- Scaffold source: `src/data/scaffolded-modules.js`

## Phase 3

## Phase 4

## Phase 5

- Investigations workspace and network visualization

## Phase 6

- Compliance and case-management models with audit trails

## Deep/dark web and restricted-source boundary

Only lawful, licensed, customer-authorized, or public sources are in scope. The project must not implement unlawful access, credential theft, malware deployment, raw password display, restricted scraping, or authentication bypass.
